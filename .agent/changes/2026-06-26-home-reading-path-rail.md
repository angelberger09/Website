# 2026-06-26 — Homepage reading path rail

## Changed

- Added a visible explanatory note to the homepage `home-path-map` so the page-level route is described as a calm chapter trail.
- Added `home-path-map__thread` markup to each homepage reading-path card so the steps visually connect as one paper trail rather than separate equal tiles.
- Added `app/home-reading-path-rail-pass.css` and loaded it from `app/layout.jsx` after the prior homepage path passes.
- Updated `app/home-reading-path-rail-pass.css` to give the path map a stronger paper-field backing, three-column desktop chapter-card layout, responsive two-column and one-column fallbacks, direct cue slips, and visible vertical thread marks.
- Updated the pages matrix for `PAGE-001`.

## Why

The homepage already had a left-docked docket and chapter fields, but the quick reading path still read partly like a row of separate small UI cards. Active lessons call for a human-readable left-docked editorial path, flattened paper hierarchy, and physical paper/collage surfaces. This pass makes the long homepage sequence visibly easier to scan without adding fake content, private material, new routes, or heavier interaction chrome.
