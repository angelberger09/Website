# Notes reader path spine

Timestamp: 2026-06-27 01:08 ET

## Interface change

Added `app/notes-path-reader-spine-pass.css` and loaded it from `app/layout.jsx` immediately after the existing Notes path direct-cue pass.

The existing Notes `After the shelf` board now has a visible vertical `reader path` paper spine, an ordered route cue, a dashed stitched thread, and numbered paper stamps on the three next-path cards.

## Why

The Notes route already had honest route-continuity copy and paper/photo cards, but the path could still read as three separate cards. This pass makes the board scan as one guided reader path from latest note to archive to front room, supporting the calm-room and human-readable editorial path goals without changing data, routes, links, or inventing content.

## Satisfaction check

- Target state: partially satisfied.
- Mismatch: Notes next-path board needed stronger composed-room guidance.
- Visible proof: the board now has a left spine, route-order label, connecting thread, and numbered card stamps.
