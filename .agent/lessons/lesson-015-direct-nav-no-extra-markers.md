# LESSON-015 — Use direct nav labels without extra markers

Date: 2026-06-23 23:50 ET  
Status: active

## Context

A screenshot review showed the current header navigation with direct labels, small edge numbers, and a separate `Pages` button still present.

## Feedback or signal

The edge numbers and extra text are not wanted. The `Pages` button should have been removed after the direct links became visible. The current visual treatment also feels too soft overall.

## Interpretation

The intended goal is direct navigation, not a labeled Pages trigger plus a revealed menu. Visible nav should use clear labels only, with no numbering system, extra helper text, or redundant `Pages` control. The visual language should keep softness in the brand, but add more confidence, contrast, and structure.

## Rule created

When direct page links are visible, remove the separate `Pages` button. Navigation labels should be plain destination names without edge numbers or extra explanatory text. Balance softness with sharper contrast and clearer hierarchy.

- COMP-002 Folder Pages Menu
- DESIGN-012 Unboxed Left-Expanding Menu
- DESIGN-015 Direct Unnumbered Navigation

## Metric

METRIC-014: Header nav should show direct destination labels only, with no edge numbers, no extra helper text, no redundant `Pages` button, and enough contrast that the UI does not feel overly soft.
