# 2026-06-26 — Notes room key focus

## Changed

- Added `app/notes-room-key-focus-pass.css`.
- Loaded the new pass from `app/layout.jsx` after the Notes section-spine pass.

## Why

The Notes index and Blog compatibility route already had paper reader boards, note contact sheets, source receipts, local path cards, and section-spine labels, but the opening shared PageIntro room key still blended into the generic route-intro treatment. Current calm-room, paper-piece, direct-label, and text-snippet guidance supports making the Notes opening key read as a stronger writing-room orientation object without changing Blog feed data, creating fake notes, or adding private material.

## Visible interface effect

On `/Website/notes/` and `/Website/blog/`, the existing PageIntro room key becomes a larger clipped paper key board with a stronger abstract paper/photo center, direct `latest first` and `return path` slips, a dashed continuation thread, and a more grounded room-shelf strip.

## Satisfaction check

- Target: Notes/Blog room-opening orientation.
- State before: partially satisfied; the route had many paper sections but the first room-key object was not visually strong enough as an opening orientation board.
- State after: partially satisfied, improved; the Notes room now opens with a clearer content-bearing writing-room key while preserving the existing reader/source/path sections.
