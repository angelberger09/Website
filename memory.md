# Website Repo Memory

## Purpose

This repo contains only the current Angels public website build.

## Architecture

- `Website-V1/` is the current Angels public site build.
- `Website-V1/` is a static multi-page Vite site for Angel Berger.
- Site identity, nav, social links, contact copy, and route content are sourced from `Website-V1/src/data/content.js`.
- Generated placeholder imagery lives separately in `Website-V1/src/data/generated-images.js`.
- Soft paper/yarn art assets live under `Website-V1/assets/`.
- `Website-V1/dist/` is generated by Vite and should not be edited by hand.
- Non-website software workspaces were migrated to the sibling `Software` repo.

## Conventions

- Keep this repo focused on the current V1 public website only.
- Do not add review dashboards, intake tooling, marketplace software, media pipelines, or validation archives to this repo.
- The Angels site should use a soft paper/yarn visual direction with warm translucent surfaces, no hard card borders, vertically scrolling background motion, and content constrained inward to avoid horizontal overflow.
- Scane and YarnObject side-scroll art in `Website-V1` uses luminance-derived alpha masks from black pixels; do not rely on source transparency for black-background yarn assets.
- The Angels page selector should be a full-screen overlay with full-width horizontal route rows.
- UI sizing should use constraint-based responsive layout with CSS Grid/Flexbox, `clamp()`, viewport units, and min/max constraints rather than fixed desktop-only dimensions.
