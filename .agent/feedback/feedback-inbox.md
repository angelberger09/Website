# Feedback Inbox

Use this file only for public-safe feedback items that need clarification before becoming lessons.

## Inbox rules

- Keep entries brief.
- Remove private details.
- Mark each item with status.
- Move resolved items into the lesson log or change log.

## Template

```md
### FEEDBACK-000 — Short title

Date: YYYY-MM-DD  
Status: needs clarification / ready to extract / converted

#### Public-safe summary

Summarize the feedback signal without private details.

#### Possible semantic items

- PAGE-000
- COMP-000

#### Next question

What should be clarified before turning this into a lesson?
```
