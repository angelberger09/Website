## Goal

Make the Notes source/readiness section scan as a connected writing-shelf path by visually binding the promise strip, source receipt ledger, and fallback receipt group.

## Boundary

Preserve the current Blog feed contract and reader behavior. Do not add fake posts, fake screenshots, private writing, raw endpoint copy, credentials, workflow logs, or new external data claims.

## Quality checks

- The source map should read as one connected paper path, not separate loose card groups.
- Source receipt cards should show a visible sequence without adding technical labels.
- Fallback rules should feel like part of the same source path.
- Cut-paper labels, thread/ribbon marks, and pin dots should remain subtle and readable.
- Mobile layout should avoid horizontal overflow and keep labels nonessential.
