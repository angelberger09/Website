# LESSON-011 — Make the header emerge from offscreen top

Date: 2026-06-23 23:30 ET  
Status: active

## Context

A screenshot review showed the current floating header sitting fully inside the viewport as a centered pill.

## Feedback or signal

The top navigation should feel like it is coming from offscreen above the page rather than floating fully within the canvas.

## Interpretation

The header should still be readable and accessible, but its placement should create a stronger sense of physical entry, depth, and studio atmosphere.

## Rule created

Treat the header as an edge-attached surface. It may be visually tucked, clipped, or partially emerging from the top edge, but the identity text and Pages control must remain usable.

## Applies to

- COMP-001 Floating Header
- DESIGN-004 Floating Header
- DESIGN-011 Offscreen Top Emergence
- INT-008 Top Edge Entry

## Metric

METRIC-010: The header should read as entering from above while all controls remain readable, clickable, keyboard-focusable, and visually intentional.
