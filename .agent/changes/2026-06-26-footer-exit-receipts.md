# 2026-06-26 — Footer exit receipt shelf

## Interface change

Added a shared footer exit-receipt shelf in `app/site-chrome.jsx` and styled it with `app/footer-exit-receipts-pass.css`, loaded from `app/layout.jsx`.

## Why

The footer already had direct route slips and a calmer room loop, but the bottom chrome still moved from the studio mark into navigation without one compact paper object that summarized the visitor path. Current calm-room, paper-snippet, direct-label, and continuity guidance supports adding a visible receipt shelf that reads as an exit path across all routes.

## Satisfaction state

Partially satisfied. The footer is already a paper trail, but this pass makes the shared exit orientation more visible and content-bearing.
