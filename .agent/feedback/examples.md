# Feedback Transformation Examples

## Example 1 — Too technical

### Feedback

"This feels too technical."

### Extracted signal

The visible site copy is prioritizing implementation details over visitor orientation.

### Lesson

Use human-facing copy first. Keep technical structure available but not dominant in public UI.

### Files to update

- `.agent/lessons/lesson-log.md`
- `.agent/matrices/feedback-lessons.matrix.md`
- `.agent/matrices/design-language.matrix.md`

## Example 2 — Navigation feels confusing

### Feedback

"I do not understand where to click."

### Extracted signal

Navigation labels or hierarchy are not clear enough.

### Lesson

Each navigation item needs a plain-language title and one-line purpose.

### Files to update

- `.agent/matrices/components.matrix.md`
- `.agent/matrices/interactions.matrix.md`
- `.agent/matrices/quality-metrics.matrix.md`

## Example 3 — Make it softer

### Feedback

"Make this softer."

### Extracted signal

### Lesson

Prefer warm neutrals, rounded structure, spacious layout, and gentle copy over sharp dashboard styling.

### Files to update

- `.agent/matrices/design-language.matrix.md`

## Example 4 — Keep it public-safe

### Feedback

"Do not put private stuff in the repo."

### Extracted signal

Public project memory must be distilled and non-sensitive.

### Lesson

Store the design or workflow lesson, not the raw vulnerable context.

### Files to update

- `.agent/feedback/public-safety-filter.md`
