# 2026-06-27 — Portfolio process runway

## Classification

- Target: Portfolio publishing rhythm section
- Satisfaction state before change: partially satisfied
- Action type: visible Website interface refinement

## In-bounds mismatch

The Portfolio publishing rhythm already had public-safe copy, honest abstract image surfaces, direct step labels, and an earlier paper-thread treatment. It still risked scanning as repeated step cards rather than one guided public-project path through the Portfolio room.

## Visible Website change

Added `app/portfolio-process-runway-pass.css` and loaded it in `app/layout.jsx` after the existing Portfolio process thread pass. The new pass turns the rhythm into a more visible publishing runway with:

- a clipped `publishing runway` paper label,
- a larger paper-field tray behind the ordered list,
- a dashed horizontal guide thread on desktop,
- staggered lifted paper/photo step slips,
- stronger photo centers and phase labels,
- a mobile-safe vertical thread fallback.

The pass preserves existing Portfolio copy, data, routes, accessibility labels, and the no-fake-project boundary.

## Why

This supports the calm-room goal by making the process section read as one composed publishing path, not a loose set of repeated UI cards. It also reinforces paper-first material, text snippets, photo-led content surfaces, flattened hierarchy, and the Portfolio archive's public-safe publishing rhythm.
