# Homepage room status ledger budget

Timestamp: 2026-06-27 16:22 ET

## Target

The homepage `Current rooms` ledger map was the newest visible room-status composition and loaded after the terminal room surface budget. Its public-room map, vertical spine, connector thread, and staggered room slips were useful, but the late pass could still read a little heavier than the calmer terminal-budgeted route surfaces.

## Satisfaction state

Partially satisfied.

The room-status section already read as a composed map instead of a loose cluster of cards, but the latest layer still needed budget alignment so the visible paper treatment stayed calm, readable, and performant.

## Visible change

Refined `app/home-room-status-ledger-map-pass.css` in place:

- added `contain`, `content-visibility`, and intrinsic sizing to the room-status section;
- reduced the vertical spine, connector thread, board, slip, and cue shadows;
- softened thread opacity and proof-cue transforms;
- reduced staggered slip rotation so the section remains readable;
- added mobile and reduced-motion fallbacks that remove transforms without removing the map structure.

## Boundary

No new content, fake room data, fake products, private material, or route behavior was added. The pass only tunes the existing homepage room-status composition.
