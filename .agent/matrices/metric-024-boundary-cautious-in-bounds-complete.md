# METRIC-024 — Boundary-Cautious, In-Bounds Complete

Timestamp: 2026-06-24 23:41 ET

## Target

Feedback-alignment runs protect boundaries first, then execute in-bounds corrections strongly enough to satisfy the chosen target.

## How to check

A run should be able to name:

1. the current `.agent` item that supports the work,
2. the boundary that limits the work,
3. the satisfaction state before acting,
4. the exact bounded surface area being changed or intentionally left unchanged,
5. whether the change resolves that surface area visibly or materially rather than symbolically.
