# Human-facing source copy pass

Timestamp: 2026-06-25 07:10 ET

## Intent

Make the rendered Website copy warmer and less implementation-facing without changing the underlying public source contracts.

## Mismatch

The active copy lessons say visible Website language should feel human-facing and studio-like before it exposes technical structure. Several rendered surfaces still used terms such as `Website-owned`, `repo`, `raw feed`, and similar internal-source phrasing. That was public-safe but too implementation-facing for visitor orientation.

## Satisfaction state

Partially satisfied before this pass.

## Website source changed

- `app/page.jsx`
- `app/site-data.js`
- `app/source-readiness.js`
- `app/notes/NotesPageClient.jsx`
- `app/store-readiness.js`

## Visible change

- Homepage route section now says `Studio rooms` and describes deeper source spaces as growing quietly behind the visitor path.
- Homepage highlights and studio-shape notes now describe connected source rhythm without naming repos or raw feeds.
- Notes intro, loading text, published-source stamp, support notes, source board heading, and readiness cards now use `studio reading door`, `public writing source`, `public notes path`, and `quiet backup path` language.
- Portfolio visible project copy now describes the studio shell and notes reader without leading with framework or repo terms.
- Store future-source rules now say the Website translates future source material into calm studio cards instead of exposing behind-the-scenes source language.

## Boundary

The pass did not invent posts, products, portfolio work, screenshots, shop links, or private material. It preserves existing public-source URLs and data fields while improving visitor-facing language.
